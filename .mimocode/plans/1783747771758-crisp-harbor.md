# OCWS zigshell-cairo-pango Architecture Redesign

## Problem Statement

The current zigshell-cairo-pango implementation has accumulated significant technical debt:
- CSS rules are defined in 3+ places per component (central stylesheet + widget `#CSS` block + theme fallback)
- The token layer is nominally the "single source of truth" but tokens.css has broken alpha variants and ocws.css bypasses tokens entirely
- Mode files duplicate widget lists instead of composing from `.set` files
- Widget variants proliferate without clear selection guidance
- The control-center reimplements sliders already in dedicated widgets

## Diagnosis: Specific Issues

### Issue 1: CSS Triplication
Every styled component has CSS in up to 3 places:

| Component | `ocws.css` | Widget `#CSS` block | `theme.css` |
|-----------|-----------|---------------------|-------------|
| dock | lines 379-448 | dock.widget lines 30-108 | lines 186-209 |
| tray | lines 453-462 | tray.widget lines 11-25 | lines 296-312 |
| show-desktop | lines 355-374 | showdesktop.widget lines 14-28 | — |
| media-player | — | media-player.widget lines 133-246 | — |
| control-center | — | ocws-control-center.widget lines 90-154 | — |

Widget `#CSS` blocks should contain ONLY widget-specific overrides that don't belong in the central stylesheet. Currently they contain full duplicates of central styles.

### Issue 2: Broken Token Layer
`tokens.css` declares alpha-variant tokens but assigns them solid colors:
```css
@define-color ocws_surface0_alpha50 #313244;  /* same as ocws_surface0 */
@define-color ocws_bg_alpha85 @theme_bg_color;  /* same as ocws_bg */
```
Meanwhile `ocws.css` bypasses tokens and references `@theme_bg_color` and hardcoded `rgba()` directly (e.g. `rgba(243, 139, 168, 0.15)` on line 305).

### Issue 3: Mode Duplication
7 modes, each hardcoding widget lists. Example:
- `single.mode` lists 10 widgets inline (lines 17-32)
- `compact.mode` lists 8 widgets inline (lines 18-49)
- `minimal.mode` lists 3 widgets inline (lines 19-29)
- `dual.mode` uses `.set` for statusbar but hardcodes dockbar

Adding a new widget (e.g. clipboard) requires editing 4+ mode files.

### Issue 4: Widget Variant Proliferation
| Widget | Variants | Unclear which to use |
|--------|----------|---------------------|
| volume | volume-text.widget | — |
| brightness | brightness-text.widget | — |
| network | network-text.widget, network-bandwidth.widget | Which? |
| cpu | cpu-text.widget, cpu-monitor.widget | Which? |
| memory | memory-text.widget, memory-monitor.widget | Which? |
| dock | dock.widget, dock-exp.widget | When to use which? |

### Issue 5: Control Center Reimplements Existing Widgets
`ocws-control-center.widget` contains its own volume slider + brightness slider + WiFi/BT toggles, duplicating:
- volume-text.widget's popup
- brightness-text.widget
- bluetooth.widget
- wifi.widget

### Issue 6: Inconsistent Naming Conventions
- `X` prefix: `XVolLevel`, `XCpuLoad`, `XMemPct` (computed)
- `OCWS` prefix: `OCWSBarThickness`, `OCWSTopSize` (env)
- No prefix: `week_starts_on_sunday` (local)
- `$` sigil for local: `$XPowerRaw`, `$value`
- Mixed: `XPowerRaw` (no sigil) vs `$XPowerRaw` (sigil) in same widget

### Issue 7: Dock Apps Hardcoded
`dock-apps.widget` hardcodes 6 apps. A `zigshell-cairo-pango-dock.json` file exists but isn't used by the widget.

### Issue 8: `theme.css` is Dead Weight
`theme.css` (357 lines) defines styles that `ocws.css` completely overrides. It only matters for the `dock-exp.css` and `island.mode` edge cases, but adds confusion about which stylesheet is authoritative.

## Proposed Redesign

### Phase 1: Eliminate Widget `#CSS` Duplicates

**What**: Remove all CSS from widget `#CSS` blocks that already exists in `ocws.css`. Widget `#CSS` blocks become empty (just the `#CSS` header) or contain only widget-specific overrides.

**Files to modify:**
- `dotfiles/ocws/widgets/core/dock.widget` — remove 80 lines of CSS (keep `#CSS` header only)
- `dotfiles/ocws/widgets/core/showdesktop.widget` — remove 15 lines of CSS
- `dotfiles/ocws/widgets/core/tray.widget` — remove 17 lines of CSS
- `dotfiles/ocws/widgets/popups/ocws-control-center.widget` — remove 64 lines of CSS, move unique styles (`.cc_row`, `.cc_icon`, `.cc_toggles`, `.cc_toggle*`) to `ocws.css`
- `dotfiles/ocws/widgets/media/media-player.widget` — move 113 lines of media popup CSS to `ocws.css`

**Keep in widget `#CSS`**: Nothing. All CSS moves to `ocws.css`.

### Phase 2: Fix Token Layer

**What**: Make `tokens.css` actually useful. Remove broken alpha variants. Ensure `ocws.css` uses tokens consistently.

**Changes to `tokens.css`:**
- Remove the 14 broken alpha-variant tokens (lines 55-68) — they're identical to their base tokens
- Add a comment explaining that `alpha()` is used at the call site, not in token definitions

**Changes to `ocws.css`:**
- Replace hardcoded `rgba(0, 0, 0, 0.4)` shadow references with a token: `@define-color shadow @ocws_bg;` + `alpha(@shadow, 0.4)` at usage — or just keep as-is since GTK CSS `alpha()` only works on color references, not `rgba()`
- Actually: GTK CSS doesn't support `rgba()` with hex — the current `rgba(0, 0, 0, 0.4)` in ocws.css line 18 is already invalid GTK CSS. Replace all `rgba()` with `alpha()` or color references.

**Audit `ocws.css` for `rgba()` usage:**
- Line 18: `rgba(0, 0, 0, 0.4)` → use token
- Line 65: `rgba(0, 0, 0, 0.4)` → use token
- Line 155: `rgba(0, 0, 0, 0.5)` → use token
- Line 218: `rgba(166, 173, 200, 0.3)` → use token
- Line 305: `rgba(243, 139, 168, 0.15)` → use `alpha(@ocws_urgent, 0.15)`
- Line 317: `rgba(0, 0, 0, 0.2)` → use token
- Lines 468-479, 515-522, 534-535: hardcoded `rgba()` in detail popup → use tokens

### Phase 3: Unify Mode System

**What**: Make all modes compose from `.set` files. No mode hardcodes widget lists.

**Changes:**
- Create `dotfiles/ocws/widget-sets/dock.set`:
  ```
  widget "dock"
  ```
- Create `dotfiles/ocws/widget-sets/taskbar.set`:
  ```
  taskbar { ... }
  ```
- Rewrite `single.mode` to use `.set` includes
- Rewrite `compact.mode` to use `.set` includes
- Rewrite `minimal.mode` to use a new `minimal.set`
- Rewrite `dual.mode` dockbar to use `.set` includes

**New `.set` files needed:**
- `widget-sets/minimal.set` — launcher, clock, volume, battery, tray
- `widget-sets/dock.set` — dock widget
- `widget-sets/taskbar.set` — taskbar config block

### Phase 4: Consolidate Control Center

**What**: Make the control center a thin wrapper that references existing widgets' variables, not a reimplementation.

**Changes to `ocws-control-center.widget`:**
- Volume slider: reference `XVolLevel` / `XVolMuted` (already does this)
- Brightness slider: reference `XBrightness` (already does this)
- WiFi/BT toggles: reference `XNetState` / `XBtState` (already does this)
- This widget is actually fine — it's a unified panel that aggregates controls. The duplication is acceptable for a "control center" UX pattern.

**Keep as-is** — the control center is a legitimate aggregation point.

### Phase 5: Eliminate `theme.css`

**What**: Remove `theme.css` entirely. `ocws.css` is the real stylesheet.

**Changes:**
- Delete `dotfiles/ocws/css/theme.css`
- Update `ocws.config` to remove `include("css/theme.css")`
- Verify no mode or widget references `theme.css` directly

### Phase 6: Fix Dock Apps Configuration

**What**: Use the existing `zigshell-cairo-pango-dock.json` instead of hardcoded buttons.

**Changes to `dock-apps.widget`:**
- Replace hardcoded buttons with a scanner that reads `zigshell-cairo-pango-dock.json`
- Or: keep hardcoded but document that users should edit this file (simpler, avoids JS complexity)

**Recommendation**: Keep hardcoded (simpler) but add a comment pointing to `zigshell-cairo-pango-dock.json` as the canonical app list for the expanded dock variant.

### Phase 7: Normalize Variable Naming

**What**: Establish and document a clear naming convention.

**Convention:**
- `X` prefix + PascalCase: computed/scanner output (`XVolLevel`, `XCpuLoad`)
- `OCWS_` prefix + SCREAMING_SNAKE: environment variables (`OCWS_BAR_THICKNESS`)
- No prefix + snake_case: local/private vars (`week_starts_on_sunday`)
- `$` sigil: only for local variable references in expressions (`$value`)

**Changes:**
- Rename `OCWSBarThickness` → `OCWS_BAR_THICKNESS` in `env.config` and all references
- Rename `OCWSTopSize` → `OCWS_TOP_SIZE`
- Rename `OCWSBottomSize` → `OCWS_BOTTOM_SIZE`
- Update `contracts/variables.ini` to reflect new names
- Update all mode files and bar configs that reference these vars

## Implementation Order

1. **Phase 2** (Fix tokens) — Foundation, no behavioral change
2. **Phase 1** (Remove widget CSS duplicates) — Big cleanup, ~200 lines removed
3. **Phase 5** (Eliminate theme.css) — Remove 357 lines of dead code
4. **Phase 7** (Normalize variables) — Rename env vars
5. **Phase 3** (Unify modes) — Create `.set` files, rewrite modes
6. **Phase 6** (Dock apps) — Minor improvement
