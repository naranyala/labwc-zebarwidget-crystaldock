## Phase 1: Shared Security Utilities (create once, use everywhere)

### 1A. Add `is_shell_safe()` to `src/libocws/ocws_string.h`

The codebase already has a local `is_shell_safe()` in `ocws-welcome.c:74-85`. Extract it into the shared header so all C files can use it without duplication.

**File:** `src/libocws/ocws_string.h`
- Add `static inline int ocws_is_shell_safe(const char *s)` — rejects `;|&$\`(){}[]`\\<>'\n\r`
- Keep the existing `ocws_is_shell_safe` name consistent

### 1B. Add safe `snprintf` wrapper for shell commands

Many `system()` calls build commands via `snprintf` with single-quote wrapping. A shared helper like `ocws_shell_escape()` that replaces `'` with `'\''` would prevent injection across all call sites.

**File:** `src/libocws/ocws_string.h`
- Add `static inline int ocws_shell_escape(char *dst, size_t dstsz, const char *src)` — escapes single quotes for safe shell interpolation

## Phase 2: CRITICAL — Command Injection (C code)

### 2.1 `src/gui/ocws-wallpaper-picker.c:30-36`
- **Issue:** Filename from file dialog interpolated into `system()` via `snprintf` with double-quote wrapping
- **Fix:** Use `fork()+execvp()` instead of `system()`. Build args array for `swaybg` directly. For `wallpaper-theme.sh`, use `fork()+execlp("wallpaper-theme.sh", ...)` with the filename as a separate arg (no shell).

### 2.2 `src/gui/ocws-welcome.c:69` — `send_notification()`
- **Issue:** `title` and `body` passed to `notify-send` via `system()` with single-quote wrapping. `'` in input → injection.
- **Fix:** Use `fork()+execlp("notify-send", ...)` with args array. Or use `ocws_shell_escape()` on title/body before interpolation.

### 2.3 `src/gui/ocws-welcome.c:178` — `on_shell_select()`
- **Issue:** `mode` passed unsanitized into `system()`.
- **Fix:** `mode` comes from a hardcoded const string array (line 176: `const char *mode = (const char *)data`), so it's safe in practice. Add a comment documenting this invariant. Alternatively, add `ocws_is_shell_safe(mode)` guard.

### 2.4 `src/gui/ocws-welcome.c:434,466,470` — `on_mount_partition()`
- **Issue:** `part` (device path from lsblk), `mount_point` (derived from disk label/UUID) passed to `system()`. Labels containing `'` break out of single-quote wrapping.
- **Fix:** Validate `part` matches `/dev/` prefix. Use `ocws_shell_escape()` on `mount_point` and `disk_name`. Or switch to `fork()+execvp()` for `mkdir`, `mount`, `pkexec`.

### 2.5 `src/gui/settings/settings-ui.c:46` — `kv_set()`
- **Issue:** `value` interpolated into `kv_set` via `snprintf` + `system()` with only single-quote wrapping.
- **Fix:** Use `ocws_shell_escape()` on `value` before interpolation. Or use `fork()+execlp()` with the kv binary.

### 2.6 `src/gui/settings/settings-ui.c:275,278,280` — slider `on_slider_changed()`
- **Issue:** `cmd_template` formatted with user slider value, then passed to `system()`.
- **Fix:** Slider values are integers/doubles from GTK scale widgets (bounded by `min`/`max`). The `cmd_template` is a compile-time constant. Add a comment documenting this safety invariant. No code change needed — the values are bounded by the widget range.

### 2.7 `src/cli/ocws-lock.c:75-81` — `start_daemon()`
- **Issue:** `exe_path` (from `/proc/self/exe`) embedded into `swayidle` command string → `system()`.
- **Fix:** `exe_path` comes from `readlink("/proc/self/exe")` which is the binary's own path — safe by definition. Add a comment. Or switch to `fork()+execvp()` for swayidle.

### 2.8 `src/gui/ocws-pkgmgr.c:250-254`
- **Issue:** Package names from `DEPS[]` interpolated into `pkexec pacman -S ... %s` → `system()`.
- **Fix:** Package names are compile-time constants in the `DEPS[]` array. Safe by construction. Add a comment documenting this.

### 2.9 `src/gui/ocws-fonts-mgr/fonts-mgr-installer.c:65`
- **Issue:** `system(cmd)` with dynamically constructed command.
- **Fix:** `cmd` is built from `tmp_path` and `install_dir` which are derived from the font package's archive name and the user's font directory. Use `ocws_shell_escape()` on both values. Or use `fork()+execlp()`.

### 2.10 `src/gui/ocws-equalizer.c:154` / `ocws-equalizer-enhanced.c`
- **Issue:** Equalizer values passed to `system()`.
- **Fix:** The equalizer applies EQ via command execution. Values come from GTK widgets (bounded doubles). Add validation (range check) and use `ocws_shell_escape()` or `fork()+execvp()`.

### 2.11 `src/cli/ocws-fonts-cli.c:217,234,236`
- **Issue:** Font package URLs and paths passed to `system()` with single-quote wrapping only.
- **Fix:** URLs are compile-time constants. Paths are derived from user config + compile-time names. Use `ocws_shell_escape()` on dynamic paths. For URLs, validate they start with `https://`.

### 2.12 `src/plugins/network/network.c:34-35`
- **Issue:** `g_interface` (from `/proc/net/dev`) interpolated into `popen()`.
- **Fix:** Validate `g_interface` against `/proc/net/dev` format (alphanumeric + `:` only). Add `ocws_is_shell_safe()` check. Or use `fork()+execvp()` with `ip` command.

### 2.13 `src/daemons/ocws-brokerd.c:61`
- **Issue:** `topic` and `value` from plugin bus passed to `execlp("ocws-emit")`.
- **Fix:** `execlp()` with separate args is already safe — no shell involved. The issue is that a malicious plugin could pass arbitrary args. Add input validation: reject topics/values containing null bytes or control characters.

## Phase 3: CRITICAL — Shell Script eval

### 3.1 `scripts/actions/launcher.sh:48` — `eval "$cmd" &`
- **Issue:** Executes whatever the user typed into the run prompt.
- **Fix:** This is a "Run Command" feature — the user intentionally types a command to execute. This is by design. Add a warning comment. Consider adding a confirmation dialog for dangerous patterns (rm, dd, etc.) or limiting to PATH-resolved commands.

### 3.2 `scripts/actions/launcher.sh:83` — `eval "$selected" &`
- **Issue:** Executes lines from `favorites.txt` as shell code.
- **Fix:** Replace `eval "$selected" &` with `"$selected" &` — this runs it as a command without shell interpretation. If the user needs shell features (pipes, etc.), they should wrap in `sh -c`. Better: change favorites format to one command per line and run via `command "$selected"`.

### 3.3 `install.sh:290,300,319,329` — curl-to-bash
- **Issue:** `bash -c "$(curl -fsSL ...)"` downloads and executes remote scripts with no integrity verification.
- **Fix:** Add checksum verification. Download to temp file, verify against known hash, then execute. Or pin to specific commit/tag. At minimum, add `--fail` to curl and check exit code before executing.

## Phase 4: HIGH — Buffer Overflows

### 4.1 `src/gui/settings/settings-ui.c:505-534` — `load_system_info()`
- **Issue:** Six consecutive `strcat(info, ...)` calls into `info[4096]`. Can overflow with long system output.
- **Fix:** Replace `strcat` chain with `snprintf(info + offset, sizeof(info) - offset, ...)` pattern. Track `offset` variable and check bounds before each append.

### 4.2 `src/gui/ocws-pkgmgr.c:239-240`
- **Issue:** `strcat(pkgs, pkg_name)` into `pkgs[2048]` inside a loop over all deps.
- **Fix:** Replace with `snprintf(pkgs + offset, sizeof(pkgs) - offset, "%s ", pkg_name)` and track offset. Break if buffer full.

### 4.3 `src/cli/ocws-search.c:90-91`
- **Issue:** `strcat(input_list, ...)` into `input_list[4096]` in a loop.
- **Fix:** Replace with `snprintf` + offset tracking. The engines array is compile-time and small (~10 entries), so overflow is unlikely but should still be guarded.

### 4.4 `src/gui/ocws-dock-mgr.c:64-89` — `detect_shell()`
- **Issue:** `strcpy(current_config.shell, ...)` and `strcpy(current_config.config_path, path)` without bounds checking.
- **Fix:** Replace `strcpy` with `strncpy(current_config.shell, "dms", sizeof(current_config.shell) - 1)` + null-terminate. Same for `config_path`. Values are short constants but defensive coding is warranted.

## Phase 5: HIGH — Integer Overflow / NULL Dereference

### 5.1 `src/cli/ocws-color.c:123`
- **Issue:** `w * h` can overflow `int`, resulting in undersized `malloc()` → heap overflow.
- **Fix:** Cast to `size_t` before multiplication: `malloc(sizeof(ColorEntry) * ((size_t)w * h / SAMPLE_STRIDE + 1))`. Also add NULL check after malloc.

### 5.2 `src/gui/ocws-dock-mgr.c:102,195,233,569`
- **Issue:** `malloc()` without NULL check, followed by `fread()` → null pointer dereference on OOM.
- **Fix:** Add NULL checks after each `malloc()`. If NULL, close file and return gracefully.

## Phase 6: HIGH — Predictable /tmp Paths (Shell Scripts)

### 6.1 `scripts/toggle-natural-scroll.sh:74,81,89,91`
- **Issue:** Writes to `/tmp/90-touchpad.hwdb` then `sudo cp`. Symlink attacker can overwrite any file.
- **Fix:** Use `mktemp` to create a temporary file: `HWDB_FILE=$(mktemp /tmp/90-touchpad-XXXXXX.hwdb)`. Clean up with `trap` on exit.

### 6.2 `scripts/actions/download-icons.sh:45-48`
- **Issue:** `cd "/tmp/$CHOSEN"` with `git clone`. Local user can pre-create as symlink.
- **Fix:** Use `mktemp -d` to create a temporary directory: `TMPDIR=$(mktemp -d /tmp/ocws-icons-XXXXXX)` with cleanup trap.

### 6.3 `scripts/applets/pomodoro.sh:9`
- **Issue:** `STATE_FILE="/tmp/ocws-pomodoro.state"`
- **Fix:** Change to `STATE_FILE="${XDG_RUNTIME_DIR:-/tmp}/ocws-pomodoro.state"` or use `mktemp`.

### 6.4 `scripts/start-redshift.sh:34,122,141,159`
- **Issue:** `PID_FILE="/tmp/redshift.pid"`
- **Fix:** Change to `PID_FILE="${XDG_RUNTIME_DIR:-/tmp}/redshift.pid"`.

### 6.5 `scripts/ocws-icon-downloader.sh:13`
- **Issue:** `DOWNLOAD_DIR="/tmp/ocws-icons"`
- **Fix:** Change to `DOWNLOAD_DIR="$(mktemp -d /tmp/ocws-icons-XXXXXX)"` with cleanup trap.

### 6.6 `scripts/install-contour.sh:28`
- **Issue:** `BUILD_DIR="/tmp/ocws-contour-build"`
- **Fix:** Change to `BUILD_DIR="$(mktemp -d /tmp/ocws-contour-build-XXXXXX)"` with cleanup trap.

## Phase 7: HIGH — Shell Script Quality

### 7.1 `scripts/actions/icon-theme-picker.sh:35,37,49,51`
- **Issue:** `$CHOSEN` from rofi used unescaped in `sed` patterns. Metacharacters break sed.
- **Fix:** Escape sed metacharacters in `$CHOSEN` before use: `ESCAPED=$(printf '%s\n' "$CHOSEN" | sed 's/[.[\/*^$]/\\&/g')`. Or use a different approach (awk, direct file write).

### 7.2 `scripts/actions/kvstore.sh:34`
- **Issue:** Non-atomic append + grep + mv. Concurrent writes corrupt data.
- **Fix:** Use `flock` for file locking: `exec 9>"$KVSTORE_FILE.lock"; flock 9`. Or use the C binary for writes (it already handles atomic writes via `ocws_kv_flush`).

### 7.3 `scripts/ocws-autorun.sh:48`
- **Issue:** `nohup $line` unquoted. Word splitting on autorun commands.
- **Fix:** Quote the variable: `nohup "$line" >> "$LOG" 2>&1 &`. But since `$line` is a full command (may include args), use `nohup sh -c "$line" >> "$LOG" 2>&1 &` to preserve shell features while avoiding word splitting.

### 7.4 `scripts/start-labwc.sh:92` — undefined array
- **Issue:** `NEW_OPTIONAL_DEPS` never declared; `set -u` crashes on modern bash.
- **Fix:** Add `NEW_OPTIONAL_DEPS=()` declaration before the loop. Check if it exists first: `if declare -p NEW_OPTIONAL_DEPS &>/dev/null; then ... fi`.

### 7.5 `scripts/actions.sh:13` — deploy target mismatch
- **Issue:** Looks for actions in `~/.local/bin/actions/` but install.sh deploys scripts to `dotfiles/scripts/actions/`.
- **Fix:** Update the path to match where install.sh actually deploys: `SCRIPT="$HOME/.config/ocws/scripts/actions/${ACTION}.sh"` or wherever install.sh puts them. Or add fallback search paths.

## Phase 8: MEDIUM — Remaining C Issues

### 8.1 `src/core/ocws-kv.c:225-243` — Atomic write race
- **Issue:** `.tmp` path is predictable, `remove()`+`rename()` fallback opens TOCTOU.
- **Fix:** Use `mkstemp()` for the temp file instead of predictable `.tmp` suffix. Use `link()` + `unlink()` pattern instead of `rename()` fallback.

### 8.2 `src/gui/ocws-dock-mgr.c` — Direct `fopen(path, "w")`
- **Issue:** No atomic writes or O_EXCL.
- **Fix:** Use `O_CREAT | O_EXCL` where appropriate. For config writes, use atomic write pattern (write to temp, rename).

### 8.3 `src/gui/ocws-pkgmgr.c:289`
- **Issue:** Predictable `/tmp/ocws-build-<pkg>` build directory.
- **Fix:** Use `mktemp -d` pattern: `BUILD_DIR=$(mktemp -d "/tmp/ocws-build-XXXXXX")`.

### 8.4 `src/cli/ocws-emit.c` — Unknown namespace passthrough
- **Issue:** Unknown namespace passes through unsanitized to `sfwbar -R`.
- **Fix:** Add validation: reject namespaces containing control characters, quotes, or backslashes. The `is_numeric` check already exists; add a similar `is_safe_namespace()` check.

### 8.5 `src/daemons/ocws-brokerd.c:401-419,481-483` — FD leaks
- **Issue:** Pipes/popen FDs without `O_CLOEXEC`, leaking into child processes.
- **Fix:** Add `O_CLOEXEC` flag to `pipe()` calls via `pipe2(fds, O_CLOEXEC)`. For `popen()`, use `pclose()` promptly (already done in most places).

### 8.6 Multiple `execlp()` calls — PATH resolution
- **Issue:** Rely on `PATH` resolution; attacker with `PATH` control substitutes binaries.
- **Fix:** Use absolute paths where possible (e.g., `/usr/bin/pkill` instead of `pkill`). For system utilities, this is a low-risk issue since PATH manipulation requires local access.

## Phase 9: HIGH — D-Bus / IPC & Plugin Loading

### 9.1 `src/daemons/ocws-osd-notify.c` / `ocws-notify.c` — D-Bus access control
- **Issue:** D-Bus methods registered with no access control. Any session bus process can call `Notify()`, `CloseNotification()`, etc.
- **Fix:** Add `org.freedesktop.DBus.Method.NoReply` or use `g_dbus_connection_register_object` with a bus name owner check. Alternatively, check `sender` against a whitelist or the current user's UID via `GetConnectionUnixUser`. This is aEDIUM-high effort fix. For now, document the threat model (session bus = same user, so risk is limited to other processes owned by the same user).

### 9.2 `src/daemons/ocws-brokerd.c:158` / `appletd.c:36` — dlopen without verification
- **Issue:** `dlopen()` from `~/.local/share/ocws/plugins/` and `$OCWS_PLUGIN_DIR`. No signature/checksum verification.
- **Fix:** Add a manifest file per plugin with a SHA-256 checksum of the .so file. Verify checksum before `dlopen()`. Or use `fstat()` to check file ownership/permissions (must be owned by current user, not world-writable).

## Phase 10: LOW — Remaining Issues

### 10.1 `src/gui/ocws-dock-mgr.c:117,139,166,186,209,513,584` — strncpy without null-termination
- **Issue:** `strncpy(..., 127)` without null-termination guarantee when source >= 127 bytes.
- **Fix:** Always null-terminate: `strncpy(buf, src, 126); buf[126] = '\0';`

### 10.2 `src/plugins/clipboard/clipboard.c:20,41,56` — JSON without escaping
- **Issue:** Clipboard content interpolated into JSON without escaping `"` or `\`.
- **Fix:** Add a simple `json_escape()` helper that escapes `"`, `\`, and control characters.

### 10.3 `src/cli/ocws-lock.c:111-112`, `ocws-shot.c:183,214`, `ocws-pkgmgr.c:407,418` — atoi()
- **Issue:** `atoi()` without validation. Returns 0 on failure.
- **Fix:** Use `strtol()` with error checking, or validate input before `atoi()`.

### 10.4 `getenv("HOME")` fallback to `/tmp`
- **Issue:** Pervasive across GUI and CLI code.
- **Fix:** Already partially fixed (fs.h uses `getpwuid()`). Audit remaining instances and apply same pattern.

### 10.5 `build-ocws-core.sh:96` — `make -j$(nproc) || true`
- **Issue:** Swallows build failures.
- **Fix:** Remove `|| true` so build errors propagate.

### 10.6 `install.sh:429,437,441,445` — `cp -r ... 2>/dev/null || true`
- **Issue:** Silences real errors.
- **Fix:** Remove `2>/dev/null || true` or use specific error handling.
