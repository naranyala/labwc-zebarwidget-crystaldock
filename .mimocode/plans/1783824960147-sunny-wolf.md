# Plan: Multi-Mode lxqt-panel Implementation

## Goal
Extend the existing OCWS shell mode system to support multiple lxqt-panel configurations, each with a different layout, plugin set, and optional zigshell-cairo-pango companion. Users can switch between modes at runtime via `toggle-shell` or the shell-mode-picker.

## Current State
- **One** lxqt-panel mode: `tworow` — lxqt-panel at top + zigshell-cairo-pango 2-row taskbar at bottom
- `dotfiles/lxqt/panel.conf` — single config file, deployed to `~/.config/lxqt/panel.conf`
- `toggle-shell` — mode dispatcher with 6 modes (doublepanel, zigshell-cairo-pango, minimal, dms, noctalia, tworow)
- `.mode` files in `dotfiles/ocws/modes/` — zigshell-cairo-pango entry points
- `dotfiles/ocws/bars/` — reusable bar definitions

## Architecture

```
dotfiles/lxqt/
├── panel.conf                          # Symlink/copied to ~/.config/lxqt/panel.conf
└── modes/
    ├── tworow.conf                     # Current: top panel (menu, taskbar, tray, clock)
    ├── classic.conf                    # Bottom panel (dock-style, macOS feel)
    ├── minimal.conf                    # Top panel (tray + clock only)
    ├── standalone.conf                 # Top panel (everything, no zigshell-cairo-pango needed)
    ├── dual-lxqt.conf                  # Two lxqt-panels: top status + bottom taskbar
    └── vertical.conf                   # Right-side vertical panel

dotfiles/ocws/modes/
├── (existing 8 .mode files)
├── lxqt-classic.mode                  # zigshell-cairo-pango top statusbar (pairs with classic.conf bottom panel)
├── lxqt-minimal.mode                  # zigshell-cairo-pango top minimal bar (pairs with minimal.conf top panel)
├── lxqt-standalone.mode               # empty/no zigshell-cairo-pango (lxqt-panel does everything)
├── lxqt-dual-lxqt.mode                # no zigshell-cairo-pango (two lxqt-panels)
└── lxqt-vertical.mode                 # zigshell-cairo-pango top statusbar (pairs with vertical.conf right panel)

scripts/
└── lxqt-panel-switcher                # Dispatcher: pick config, kill panel, restart
```

## Implementation Steps

### Step 1: Create lxqt-panel config variants
**Files to create:** `dotfiles/lxqt/modes/*.conf`

Each config is a complete `panel.conf` with `[General]` + `[panel1]` (and `[panel2]` for dual-lxqt). The `preferred_backend=labwc:wlroots` line is shared across all.

Key differences between configs:
- `position=` — top, bottom, right
- `plugins=` — semicolon-separated plugin list
- `size=` — panel thickness (28-48px)
- `alignment=` — left, center, right
- Per-plugin sections with tuned settings

### Step 2: Create zigshell-cairo-pango companion .mode files
**Files to create:** `dotfiles/ocws/modes/lxqt-*.mode`

Each `.mode` file is a minimal zigshell-cairo-pango config that provides whatever lxqt-panel doesn't cover:
- `lxqt-classic.mode` — top statusbar (launcher, workspaces, clock, tray, system metrics)
- `lxqt-minimal.mode` — top bar with launcher, workspaces, control-center
- `lxqt-standalone.mode` — empty/disabled (no zigshell-cairo-pango needed)
- `lxqt-dual-lxqt.mode` — empty/disabled
- `lxqt-vertical.mode` — top statusbar (launcher, workspaces, clock, system metrics)
- `lxqt-bottom.mode` — top statusbar (launcher, workspaces, clock, system metrics)

### Step 3: Create the lxqt-panel-switcher script
**File to create:** `scripts/lxqt-panel-switcher`

```
Usage: lxqt-panel-switcher <mode>
Modes: tworow, lxqt-classic, lxqt-minimal, lxqt-standalone, lxqt-dual-lxqt, lxqt-vertical, lxqt-bottom
```

Logic:
1. Validate mode name
2. Kill existing lxqt-panel processes
3. Wait for layer-shell surface release (0.3s)
4. Copy `dotfiles/lxqt/modes/<mode>.conf` → `~/.config/lxqt/panel.conf`
5. Start lxqt-panel with env vars (`QT_WAYLAND_SHELL_INTEGRATION=layer-shell`, `XDG_CURRENT_DESKTOP=labwc:wlroots`)
6. For dual-lxqt mode, panel.conf contains both [panel1] and [panel2]

### Step 4: Integrate into toggle-shell
**File to modify:** `scripts/toggle-shell`

- Add new modes to `MODES` associative array:
  - `["lxqt-classic"]="labwc + lxqt-panel bottom (classic desktop) + zigshell-cairo-pango top status"`
  - `["lxqt-minimal"]="labwc + lxqt-panel top (tray+clock) + zigshell-cairo-pango minimal"`
  - `["lxqt-standalone"]="labwc + lxqt-panel standalone (no zigshell-cairo-pango)"`
  - `["lxqt-dual-lxqt"]="labwc + dual lxqt-panels (top+bottom, no zigshell-cairo-pango)"`
  - `["lxqt-vertical"]="labwc + lxqt-panel right (vertical) + zigshell-cairo-pango top status"`
  - `["lxqt-bottom"]="labwc + lxqt-panel bottom + zigshell-cairo-pango top status"`

- Add case branches for each new mode:
  - Kill existing shells (zigshell-cairo-pango, lxqt-panel, zigshell-cairo-pango, etc.)
  - Call `lxqt-panel-switcher <mode>` for lxqt-panel config
  - Start companion zigshell-cairo-pango with the matching `.mode` file (if any)
  - Write mode to `~/.config/ocws/mode`

### Step 5: Update autostart
**File to modify:** `dotfiles/labwc/autostart`

- Add new modes to the fallback `case` block (lines 97-124)
- Each new lxqt mode calls `lxqt-panel-switcher` + starts companion zigshell-cairo-pango

### Step 6: Update install.sh
**File to modify:** `install.sh`

- Add new lxqt modes to the install menu (option 5 expanded or new options)
- Deploy `dotfiles/lxqt/modes/` directory during install

### Step 7: Update shell-mode-picker.sh
**File to modify:** `scripts/shell-mode-picker.sh`

- Add new lxqt modes to the rofi picker menu

## Verification
1. **Config validity:** Each `.conf` file should be parseable by lxqt-panel (valid INI format, all plugin types exist)
2. **Mode switching:** Run `toggle-shell lxqt-classic` (etc.) and verify:
   - Old panel killed
   - New panel started at correct position with correct plugins
   - Companion zigshell-cairo-pango started (if applicable)
   - `~/.config/ocws/mode` updated
3. **Autostart:** Set `LABWC_SHELL_MODE=lxqt-classic` and verify correct startup
4. **No regressions:** Existing `tworow` mode still works identically
