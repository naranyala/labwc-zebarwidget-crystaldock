# Plan: zigshell-blend2d

## Context

The existing Zig shell at `src/shells/zigshell-cairo-pango/` uses Cairo + Pango + librsvg for rendering to Wayland SHM buffers. The user wants to replace this with **Blend2D** — a JIT-accelerated 2D vector engine that replaces all three (Cairo + Pango + librsvg) in one library.

Sokol (GPU-based) is deferred — it requires a fundamentally different pipeline (GPU texture + CPU readback) and can't preserve layer-shell behavior cleanly.

## Architecture Reference (zigshell-cairo-pango)

The existing shell has this structure:
- `main_shell.zig` — Wayland globals, event loop, two layer surfaces (panel top + dock bottom)
- `panel.zig` — Widget system (13 widget types with measure/draw/update/click callbacks)
- `dock.zig` — Dock rendering (icon layout, hover detection, focus indicators)
- `icon.zig` — Desktop file lookup, PNG/SVG loading via librsvg, icon cache
- `toplevel.zig` — ToplevelInfo struct and array operations
- `c.zig` — `@cImport` of `dock_c.h`
- `dock_c.h` — Forward-declared C types (Cairo, Pango, librsvg) + real Wayland headers
- `dock_c_impl.c` — Real C includes for linking
- Protocol `.c/.h` files — Generated by wayland-scanner

Rendering pipeline: `ensureBuffer()` creates SHM fd → mmap → `cairo_image_surface_create_for_data()` wraps the buffer → Cairo draws directly on it → `wl_surface_attach()` submits.

---

## Strategy

Replace Cairo/Pango/librsvg with Blend2D. Blend2D renders to a `BLImage` (PRGB32 format = premultiplied ARGB8888), which maps directly to Wayland SHM format. Blend2D has built-in font support (no Pango needed) and built-in image codecs (PNG/JPEG/BMP/QOI — no librsvg needed for PNG icons).

**Key insight:** Blend2D's `BLImage` data pointer can be used as the SHM buffer directly — no pixel copying needed. `bl_image_make_mutuable()` gives access to the raw pixel data.

## Files to create

### 1. `build.zig`
Similar to cairo-pango's but links Blend2D instead of Cairo/Pango/librsvg/glib/gobject:
```
Dependencies:
  - wayland-client (system pkg-config)
  - Blend2D (built from source via CMake subproject or vendored)
  - asmjit (Blend2D's JIT dependency, optional — Blend2D has software fallback)
Protocol sources: same 3 .c files (layer-shell, toplevel, xdg-shell)
```

Blend2D build integration options:
- **Option A (recommended):** FetchContent/add_subdirectory in Zig's CMake integration
- **Option B:** Vendor Blend2D source in a `deps/blend2d/` directory
- **Option C:** Pre-build Blend2D as a static library, link from `zig-out/`

### 2. `src/c.zig`
Same pattern — `@cImport` of `dock_c.h`

### 3. `src/dock_c.h`
Replace Cairo/Pango/librsvg forward declarations with Blend2D types:

Key Blend2D C API functions needed:

- `bl_context_set_fill_style_rgba32()` — set fill color
- `bl_context_fill_text_d()` — draw text (needs BLFont)
- `bl_font_load_from_file()` — load font from .ttf file
- `bl_font_create_from_data()` — create font from memory buffer
- `bl_font_manager_init()` / `bl_font_manager_add_font()` — font management
- `bl_image_read_from_file()` — load PNG/JPEG/BMP images
- `bl_image_blit_image()` — draw one image onto another
- `bl_image_data()` — get raw pixel data pointer
- `bl_image_bytes_per_row()` — get stride

```zig
    ctx: c.BLContextCore,
    font_manager: c.BLFontManagerCore,

    pub fn begin(self: *BlendRenderer) !void;
    pub fn end(self: *BlendRenderer) void;
    pub fn fillRect(self: *BlendRenderer, x: f64, y: f64, w: f64, h: f64, color: u32) void;
    pub fn fillGradientRect(self: *BlendRenderer, x: f64, y: f64, w: f64, h: f64, c1: u32, c2: u32) void;
    pub fn drawText(self: *BlendRenderer, text: []const u8, x: f64, y: f64, font_size: f64, color: u32) void;
    pub fn measureText(self: *BlendRenderer, text: []const u8, font_size: f64) TextMetrics;
    pub fn drawImage(self: *BlendRenderer, img_path: []const u8, x: f64, y: f64, w: f64, h: f64) void;
    pub fn drawRoundedRect(self: *BlendRenderer, x: f64, y: f64, w: f64, h: f64, radius: f64, color: u32) void;
    pub fn getPixelData(self: *BlendRenderer) ?[*]u8;
    pub fn getStride(self: *BlendRenderer) i32;
};
```

### 6. `src/icon.zig` (modified from cairo-pango version)
Replace librsvg SVG loading with Blend2D's built-in image loading:
- `bl_image_read_from_file()` reads PNG/JPEG/BMP/QOI directly
- No need for desktop file lookup changes — just swap the image loading backend
- Keep the same icon cache structure
- **SVG limitation:** Blend2D does NOT have SVG loading. Fallback: PNG-only + colored circle for SVG-only icons

### 7. `src/panel.zig` (modified)
Change widget draw callbacks from `*c.cairo_t` to `*BlendRenderer`:
- `widgetText()` — use `renderer.drawText()` instead of Pango
- `widgetIconGlyph()` — use `renderer.drawText()` with larger font
- All widget draw functions receive the renderer instead of cairo_cr
- `draw_fn` signature changes: `fn(*Widget, *c.cairo_t, i32, i32, i32) void` → `fn(*Widget, *BlendRenderer, i32, i32, i32) void`

### 8. `src/dock.zig` (modified)
Change dock draw from `*c.cairo_t` to `*BlendRenderer`:
- `draw()` — use Blend2D for gradient background, icon rendering, focus bars
- Icon drawing uses `renderer.drawImage()` or `bl_image_blit_image()`

### 9. `src/main_shell.zig` (modified)
Replace SurfaceState's cairo fields with BlendRenderer:
```zig
const SurfaceState = struct {

    height: i32 = 0,
```

## Rendering pipeline (Blend2D)

```
ensureBuffer():
  1. Create SHM fd + mmap → pixel_data
  2. bl_image_init_as(&image, w, h, BL_FORMAT_PRGB32)
  3. bl_image_set_data(&image, pixel_data, stride)  ← zero-copy!
  4. bl_context_init_as(&ctx, &image, NULL)

renderPanel():
  1. bl_context_begin(&ctx) — begin recording
  2. fillRect(), drawText(), drawImage() — record commands
  3. bl_context_end(&ctx) — flush to image (JIT if enabled)
  4. image.data IS the SHM buffer → wl_surface_attach()

No pixel copying! Blend2D renders directly into the mmap'd SHM buffer.
```
