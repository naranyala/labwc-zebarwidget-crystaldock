# ZIGSHELL-CAIRO-PANGO Text Rendering and Icon Resolver Lessons

## The Problem
When launching `zigshell-cairo-pango`, the UI suffered from complete text rendering failures (missing fonts, sizes, and colors) and an inability to resolve custom status icons (rendering blank boxes or fallbacks).

## The Cause
This was traced to two completely separate configuration bugs breaking different layers of the GTK rendering engine:

1. **Text Rendering Failure (GTK CSS Crash)**:
   The main layout CSS template (`templates/ocws.css.tmpl`) included a remote font import:
   `@import url("https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap");`
   The GTK3 CSS engine strictly does not support remote HTTP `@import` URLs. When it encountered this line, it threw a fatal parsing error and completely aborted loading the rest of the CSS file, resulting in unstyled text elements.

2. **Icon Resolver Failure (Theme Override)**:
   The main `zigshell-cairo-pango` config (`dotfiles/ocws/ocws.config`) contained a hardcoded GTK theme override:
   `Theme = "Adwaita-dark"`
   By hardcoding this, `zigshell-cairo-pango` forcefully instructed GTK to use `Adwaita-dark`, completely bypassing the custom `Catppuccin-Mocha` GTK configuration generated by `theme-engine.sh` (which properly sets `IconTheme=Papirus-Dark`). As a result, `zigshell-cairo-pango` attempted to load status icons from the default Adwaita icon set instead of the custom Papirus set.

## The Solution
- Removed the invalid `@import url(...)` from the CSS template. Custom fonts (like Inter) must be installed locally on the system via the package manager.
- Removed the hardcoded `Theme = "Adwaita-dark"` line from `ocws.config`, allowing `zigshell-cairo-pango` to gracefully inherit the active system GTK settings (and by extension, the correct Icon Theme).
